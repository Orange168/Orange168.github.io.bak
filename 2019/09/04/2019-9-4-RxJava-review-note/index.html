    Create

    create

defer（订阅重新创建）/just
from
interval（1，TimeUnit.SECONDES）
Observable.just(1).repeat(5);#重复发送5数

    Transforming Observables

    buffer –> Window 返回的是Observe

    buffer(3) #缓存3个统一发出
buffer(2,3)#接收1,2,3,4,5,6->发送1,2    4,5(缓存2个，skip第三个)
buffer(3,TimeUnit.SECONDS) #缓存3s,统一发出

    FlatMap –> flatMapIterable 顺序输出

    Observable.just(1, 2, 3, 4, 5, 6, 7, 8, 9)
        .flatMap(integer -> Observable.just( integer));
# 1, 2, 3, 4, 5, 6, 7, 8, 9

    Map、Cast

    • switch()和flatMap() switch 会丢弃旧数据

    Scan 上一次的结果给下一次使用，类似于递归

    Filtering

    • debounce (500,Unit) 两个500 内有多给发射取最后一个
      throttleWithTimeOut 限流,debounce可以根据函数限流（被限流的数据会被丢弃）
    • Distinct和DistinctUtilChanged
    • ElementAt、Filter
    • First、Last
    • Skip(2) 跳过前2个、Take(2) 取前2个
    • throttleFirst(1000, TimeUnit.MILLISECONDS) 1秒中最先到达的数据
    • sample(1000, TimeUnit.MILLISECONDS) 1秒中最后到达的数据

      Combining

    • CombineLatest 2~9个Observable每个都发射一个然后组装 【直到最后一个发出才组装】
      Observable.combineLatest(createObserver(1), createObserver(2), (num1, num2) -> {
      log("left:" + num1 + " right:" + num2);
      return num1 + num2;
  }
    • Join/groupJoin Observable（时间窗内有效）
      // 分别匹配
private Observable<String> createObserver2() {
  return Observable.create(new ObservableOnSubscribe<String>() {
    @Override public void subscribe(ObservableEmitter<String> emitter) throws Exception {
      for (int i = 1; i < 5; i++) {
        emitter.onNext("Right-" + i);
        try {
          Thread.sleep(1000);
        } catch (InterruptedException e) {
          e.printStackTrace();
        }
      }
    }
  });
}
Observable.just("Left-","L2").join(createObserver2(),
      integer -> Observable.timer(3000, TimeUnit.MILLISECONDS),//源Observable
      integer -> Observable.timer(1000, TimeUnit.MILLISECONDS),//加入Ob
      (i, j) -> i + j
  ).subscribe(new Consumer<String>() {
    @Override public void accept(String s) throws Exception {
      Log.e("TAG","join:" + s + "\n");
    }
  });
    • Merege 多个发射整合，有交错（没交错cconcat）/ mergeDelayError 合并完成才发送错误
    • StartWith、Switch（多为一个Observable，后面覆盖前面） 
      Observable.just(1, 2, 3).startWith(-1, 0)
//输出：-1,0,1,2,3 可以插入list Observable
    • Zip，zipWith 按顺序，每个数据只能组合一次，发射次数由最少的决定
      //只发射了2次
createObserver(2).zipWith(createObserver(3), (s, s2) -> s + "-" + s2);
//只发射2次
Observable
  .zip(createObserver(2), createObserver(3), createObserver(4), (s, s2, s3) -> s + "-" + s2 + "-" + s3);

      Error Handling

      一、Catch

    • OnErrorReturn 返回预先值
    • OnErrorResume 返回另一个Observable
    • OnExceptionResumeNext 如果是Exception（同Resume），否专递给Subscriber

      Retry ，RetryWhen

      尝试了几次还是产生错误后，retry会将错误分发给观察者，而retryWhen会正常结束，并不会讲错误分发出去
      createObserver().retry(2)
createObserver().retryWhen(observable -> observable.zipWith(Observable.just(1, 2, 3),
          (throwable, integer) -> throwable.getMessage() + integer)
          .flatMap(throwable -> {
              log(throwable);
              return Observable.timer(1, TimeUnit.SECONDS);
          }))

      Utility

    • toList : 把数据转换成 List 集合
      Observable
        .just(1, 2, 3, 4)
        .toList()
        .subscribe(new Consumer<List<Integer>>() {
             @Override
             public void accept(List<Integer> integers) throws Exception {
               Log.e("zhao", "accept: " + integers);
         }
   });
    • Delay / DelaySubscription
    • Doxxx

      Observable的生命周期的各个阶段加上一系列的回调监听，当Observable执行到这个阶段的时候，这些回调就会被触发

    • Meterialize 将OnNext/OnError/OnComplete都转化为一个Notification
    • TimeInterval 拦截发射/取两次间隔时间 <==> TimeStamp 为每次发射打时间戳

      stringTimed.value() 获得onNext()的值

    • Timeout 没次重置，超时抛出超时异常
    • using 限时销毁资源

      Conditional and Boolean

    • All(全true 否 false )/Amb (至多9个Observable，先发先用其他丢弃)
    • Contains、IsEmpty
    • DefaultIfEmpty(10) 
      public void call(Subscriber<? super Integer> subscriber) {
          //没有调用onNext()，返回 DefaultIfEmpty：10
          subscriber.onCompleted();
}
    • SequenceEqual Obd对比 数据，序列，结束状态相同
    • TakeUntil、TakeWhile、TakeUntil、TakeWhile
      Observable
.interval(1, TimeUnit.SECONDS)
.skipUntil(Observable.timer(3, TimeUnit.SECONDS)) //可以加时间

      Aggregate

    • Concat vs merge vs startWith
    • Count Observable 没有发送完不触发
    • Reduce（类似Scan，单只输出最后结果）、Collect(容器，)
      Observable.from(list)
.collect(() -> new ArrayList<>(), (integers, integer) -> integers.add(integer));
}

      背压

      错误：MissingBackpressureException, 直接用Observable默认处理不了的存在内存中，容易导致内存溢出

      Flowable.interval( 1 , TimeUnit.MILLISECONDS)
//onBackpressureDrop 一定要放在 interval 后面否则不会生效
      .onBackpressureDrop() 
      .subscribeOn(Schedulers.io())
      .observeOn(Schedulers.newThread())
      .subscribe(new Consumer<Long>() {
            @Override
             public void accept(Long aLong) throws Exception {
                 Thread.sleep(1000);
                 Log.e("zhao", "onNext: " + aLong);
             }
     });
    • onBackpressureDrop
    • onBackpressureLatest
    • onBackpressureBuffer( 1000 ) //设置缓冲队列大小为 1000

    常用类详解

    CompositeDisposable

    CompositeDisposable Disposable 的管理类，类似于list

    CompositeDisposable.add(mDisposable ) // 得到
CompositeDisposable.clear() //退出时

    Subject

    • PublishSubject 通知每个观察者
      PublishSubject<String> publishSubject = PublishSubject.create();
publishSubject .onNext(s.toString()); //触发
    • BehaviorSubject 最后一次onNext()缓存给下一个观察者
      BehaviorSubject<Integer> behaviorSubject = BehaviorSubject.create();
    • AsyncSubject onComplete()前其他操作都会被缓存

    subscribe

    Single

    onSubscribe (onSuccess | onError)?

    一个只包含一个item或error的流

    Completable

    onSubscribe (onComplete | onError)?

    只包含completion或error信号的流

    io.reactivex.Maybe

    只包含一个maybe value或者error的流

    onSubscribe (onSuccess | onError | onComplete)?

    • Flowable request
      0..N个流，支持Reactive-Streams和背压(backpressure)

      Flowable<Integer> upstream = Flowable.create(new FlowableOnSubscribe<Integer>() {
         @Override
         public void subscribe(FlowableEmitter<Integer> emitter) throws Exception {}
     }, BackpressureStrategy.ERROR); //增加了一个参数

   Subscriber<Integer> downstream = new Subscriber<Integer>() {
         @Override
         public void onSubscribe(Subscription s) {
             Log.d(TAG, "onSubscribe");
             s.request(Long.MAX_VALUE);  //注意这句代码
         }  
         .......
     };
upstream.subscribe(downstream);
